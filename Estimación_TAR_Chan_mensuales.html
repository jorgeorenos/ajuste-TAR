Proceso de ajuste del modelo TAR para la estimaci√≥n del efecto Pass through

El presente apartado tiene como finalidad detallar el proceso de ajuste del modelo TAR utilizado para la estimaci√≥n del efecto pass-through del tipo de cambio para Guatemala por lo que se omite la explicaci√≥n del modelo en cuesti√≥n y se concentra √∫nicamente en el c√≥digo y proceso de estimaci√≥n.

El punto de partida para el ajuste del modelo TAR es conocer los umbrales que separan cada uno de los reg√≠menes inflacionarios. Debido al hecho que estos se desconocen el primer paso consiste en la estimaci√≥n de estos. El procedimiento a seguir ser√° el propuesto por Chan(1993) el cual requiere de 3 pasos a seguir.

  1. Se ordenan de menor a mayor las observaciones de la variable que define el estado del sistema y se procede a eliminar el 30% de las observaciones extremas1. Este ser√° el conjunto de umbrales posibles.

  2. Se ajusta el modelo TAR te√≥rico por cada uno de los umbrales y se captura de cada modelo la suma de residuos al cuadrado SSR.

  3. Las SSR son funci√≥n del umbral utilizado para el ajuste del modelo por lo que se grafican SSR contra umbrales obteniendo as√≠ la funci√≥n SSR. Si uno de los umbrales dentro del conjunto es el verdadero umbral del modelo la SSR disminuir√° en la meda en la cual se aproxima al umbral. Si hay m√°s de un umbral la funci√≥n SSR tendr√° tantos m√≠nimos como umbrales a considerar dentro del modelo.

Luego de estimados los umbrales se procede a ajustar el modelo TAR con los umbrales seleccionados.

Modelo te√≥rico inicial

El modelo te√≥rico es una descomposici√≥n factorial de la inflaci√≥n de la siguiente forma:

\[\Delta\%P_t = \beta_0 + \beta_1\Delta\%P_{t-1} + \beta_2\Delta\%P_t^{EUA} + \beta_3\%Y^{gap} + \beta_4\Delta\%e_t + I_t\delta\Delta\%e_t\]

donde:

\(\Delta\%P_t\) Tasa de variaci√≥n del √çndice de Precios al Consumidor de Guatemala

\(\beta_0\) Componente aut√≥nomo de la inflaci√≥n

\(\Delta\%P_{t-1}\) Componente inercial de la inflaci√≥n

\(\Delta\%e_t\) Tasa de variaci√≥n del tipo de cambio (tasa de depreciaci√≥n del tipo de cambio)

\(\Delta\%P_{t}^{EUA}\) Tasa de variaci√≥n del √çndice de Precios al Consumidor de Estados Unidos

\(Y_t^{brecha}\) Brecha relativa del producto respecto del potencial

\(I_t\) Variable dummy que es 1 si \(\Delta\%P_{t-1}<\tau\) donde \(\tau\) es el umbral y 0 en otro caso

El modelo considera a la infalci√≥n inercial (\(\Delta\%P_{t-1}\)) como la variable que define el estado del sistema, adem√°s de ello el uso de la variable dummy orgina dos tipos de reg√≠menes uno de inflaci√≥n ‚Äòbaja‚Äô y otro de inflaci√≥n ‚Äòalta‚Äô.

Estimaci√≥n de los umbrales de inflaci√≥n

A continuaci√≥n, se presenta el c√≥digo con el cual se cre√≥ el conjunto de umbrales posibles as√≠ como el ajuste del modelo TAR especificado anteriormente por cada umbral posible. El n√∫mero total de modelos estimados fue de 180.

# Creaci√≥n de todos los posibles umbrales
# N√∫mero de umbrales fuera por debajo o arriba de la muestra

umbrales_fuera <- length(inercial) - round(dim(datos)[1] - dim(datos)[1]*0.3, digits = 0)

posibles <- posibles[round((umbrales_fuera/2 + 1), 0):
                       (length(inercial)-round(umbrales_fuera/2,0))] 

# Finalmente posibles es el vector que contiene el conjunto de umbrales posibles utilizados para estimar el modelo

# Estimaci√≥n de los 180 modelos

modelos <- list()
bases <- list()

# Ciclo para estimar y nombrar los modelos

for (i in c(1:length(posibles))) {
  
  # Se genera un nombre para cada modelo a fin de poder tener una correcta identificaci√≥n
  
  nombre_modelos[i] <- c(sprintf("modelo_%s_%s", i, posibles[i]))
  
  # Se genera una base de datos para cada modelo
  # en esta base de datos d es la variable dummy que separa los reg√≠menes inflacionarios
  
  m <- mutate(datos_modelo, 
                     d = ifelse(inercial<posibles[i], yes = 0, no = 1))
  
  # Bases es la lista que almacena las bases de datos utilizadas
  # Aunque era posible estimar los modelos sin almacenar las bases de datos se almacenaron
  # para tener una ferencia de ellas
  
  bases[[i]] <- m
  
  modelo <- lm(bases[[i]],
                      formula = inf_gt_intermensual ~ 1 +
                        deprec*d + inercial +
                        inf_eua_intermensual + var_brecha - d)
  
  modelos[[i]] <- modelo
  
}

# Se coloca el nombre a cada base de datos y modelo

names(modelos) <- nombre_modelos
names(bases) <- nombre_modelos

Luego de la estimaci√≥n de los modelos se procede al c√°lculo de la SSR de cada modelo.

# SSR es el vector que contiene la suma de residuos al cuadrado de cada modelo

ssr <- c()

for (i in c(1:length(posibles))) {
  
  ssr[i] <- sum(modelos[[i]]$residuals^2)
  
}

\[\beta_i = \frac{cov_{x_iy}}{var_{x_i}}\]
